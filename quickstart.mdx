This guide will walk you through getting started with Cognia, from creating your account to organizing your first piece of content.

## Create your account

<Steps>
  <Step title="Sign up for a new account">
    Create a new account by sending a POST request to the signup endpoint with your desired username and password.

    ```bash
    curl -X POST https://api.cognia.com/api/v1/signup \
      -H "Content-Type: application/json" \
      -d '{
        "username": "your-username",
        "password": "your-secure-password"
      }'
    ```

    **Response:**
    ```json
    {
      "message": "User signed up successfully",
      "userId": "507f1f77bcf86cd799439011"
    }
    ```

    <Note>
      Your password will be securely hashed using bcrypt before being stored in the database.
    </Note>
  </Step>

  <Step title="Sign in to get your authentication token">
    After creating your account, sign in to receive your JWT authentication token.

    ```bash
    curl -X POST https://api.cognia.com/api/v1/signin \
      -H "Content-Type: application/json" \
      -d '{
        "username": "your-username",
        "password": "your-secure-password"
      }'
    ```

    **Response:**
    ```json
    {
      "message": "Login successful",
      "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    }
    ```

    <Warning>
      Keep your authentication token secure. You'll need to include it in the Authorization header for all subsequent requests.
    </Warning>
  </Step>

  <Step title="Add your first content">
    Now you can start adding content to your second brain. Cognia supports multiple content types: YouTube videos, Twitter links, web links, and notes.

    <CodeGroup>
    ```bash YouTube Video
    curl -X POST https://api.cognia.com/api/v1/content \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer YOUR_TOKEN" \
      -d '{
        "title": "Introduction to React Hooks",
        "link": "https://www.youtube.com/watch?v=dpw9EHDh2bM",
        "type": "youtube",
        "tags": ["react", "tutorial"]
      }'
    ```

    ```bash Twitter Link
    curl -X POST https://api.cognia.com/api/v1/content \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer YOUR_TOKEN" \
      -d '{
        "title": "Interesting thread on Web Performance",
        "link": "https://twitter.com/user/status/123456789",
        "type": "twitter",
        "tags": ["performance"]
      }'
    ```

    ```bash Web Link
    curl -X POST https://api.cognia.com/api/v1/content \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer YOUR_TOKEN" \
      -d '{
        "title": "MDN Documentation on Promises",
        "link": "https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise",
        "type": "link",
        "imageUrl": "https://example.com/thumbnail.jpg",
        "tags": ["javascript", "docs"]
      }'
    ```

    ```bash Note
    curl -X POST https://api.cognia.com/api/v1/content \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer YOUR_TOKEN" \
      -d '{
        "title": "Meeting Notes - Q1 Planning",
        "content": "# Meeting Notes\n\n- Discussed roadmap for Q1\n- Key priorities: performance and UX",
        "type": "note",
        "tags": ["meetings", "planning"]
      }'
    ```
    </CodeGroup>

    **Response:**
    ```json
    {
      "message": "content added",
      "createdContent": {
        "_id": "507f1f77bcf86cd799439011",
        "title": "Introduction to React Hooks",
        "link": "https://www.youtube.com/watch?v=dpw9EHDh2bM",
        "type": "youtube",
        "userId": "507f191e810c19729de860ea",
        "tags": ["react", "tutorial"],
        "createdAt": "2026-03-04T10:30:00.000Z",
        "updatedAt": "2026-03-04T10:30:00.000Z"
      }
    }
    ```

    <Note>
      **Content type requirements:**
      - `youtube`, `twitter`, `link`: Require a `link` field
      - `note`: Requires a `content` field
      - All types require `title` and `type` fields
    </Note>
  </Step>

  <Step title="Create a workspace to organize your content">
    Organize your content by creating workspaces. Each workspace can have a custom name, color, and icon.

    ```bash
    curl -X POST https://api.cognia.com/api/v1/workspaces \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer YOUR_TOKEN" \
      -d '{
        "name": "Web Development",
        "color": "#3B82F6",
        "icon": "code"
      }'
    ```

    **Response:**
    ```json
    {
      "workspace": {
        "_id": "507f1f77bcf86cd799439011",
        "name": "Web Development",
        "color": "#3B82F6",
        "icon": "code",
        "userId": "507f191e810c19729de860ea",
        "createdAt": "2026-03-04T10:30:00.000Z"
      }
    }
    ```

    You can then add content to a workspace by including the `workspaceId` when creating content:

    ```bash
    curl -X POST https://api.cognia.com/api/v1/content \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer YOUR_TOKEN" \
      -d '{
        "title": "React Best Practices",
        "link": "https://react.dev/learn",
        "type": "link",
        "workspaceId": "507f1f77bcf86cd799439011"
      }'
    ```
  </Step>

  <Step title="Search your content">
    Search across all your saved content by title, content, or links.

    ```bash
    curl -X GET "https://api.cognia.com/api/v1/search?q=react" \
      -H "Authorization: Bearer YOUR_TOKEN"
    ```

    **Response:**
    ```json
    {
      "results": [
        {
          "_id": "507f1f77bcf86cd799439011",
          "title": "Introduction to React Hooks",
          "link": "https://www.youtube.com/watch?v=dpw9EHDh2bM",
          "type": "youtube",
          "tags": ["react", "tutorial"],
          "createdAt": "2026-03-04T10:30:00.000Z"
        },
        {
          "_id": "507f1f77bcf86cd799439012",
          "title": "React Best Practices",
          "link": "https://react.dev/learn",
          "type": "link",
          "workspaceId": "507f1f77bcf86cd799439011",
          "createdAt": "2026-03-04T10:35:00.000Z"
        }
      ]
    }
    ```

    The search performs case-insensitive pattern matching across title, content, and link fields.
  </Step>
</Steps>
