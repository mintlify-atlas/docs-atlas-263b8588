## Full-text search

### Search API

The search endpoint performs case-insensitive regex matching:

```typescript
GET /api/v1/search?q=javascript
```

### Backend implementation

```typescript
app.get("/api/v1/search", userMiddleware, async (req, res) => {
  try {
    const userId = req.userId;
    const q = req.query.q as string;

    if (!q || q.trim() === "") {
      return res.json({
        results: []
      });
    }

    const searchRegex = new RegExp(q.trim(), "i");

    const results = await contentModel.find({
      userId,
      $or: [
        { title: searchRegex },
        { content: searchRegex },
        { link: searchRegex }
      ]
    });

    return res.json({
      results: results
    });
  }
  catch (error) {
    console.error("Search error:", error);
    res.status(500).json({
      message: "Internal Server error",
    })
  }
})
```

<Note>
  Search automatically filters results to only show content belonging to the authenticated user.
</Note>

### Search fields

The search query matches against three fields:

1. **Title**: The content's title
2. **Content**: The body text (for notes)
3. **Link**: The URL (for YouTube, Twitter, and web links)

### Case-insensitive matching

The regex pattern uses the `i` flag for case-insensitive search:

```typescript
const searchRegex = new RegExp(q.trim(), "i");

// Matches:
// "JavaScript" → matches "javascript", "JAVASCRIPT", "JavaScript"
// "react hooks" → matches "React Hooks", "react hooks", "REACT HOOKS"
```

For optimal search performance, the content schema includes text indexes:

```typescript

  content: "text",
```

## Filtering by content type

### Type filter API

Filter content by type using query parameters:

```typescript
GET /api/v1/content?type=note
GET /api/v1/content?type=youtube
GET /api/v1/content?type=twitter
GET /api/v1/content?type=link
```

### Backend filtering

```typescript
app.get("/api/v1/content", userMiddleware, async (req, res) => {
  try {
    const userId = req.userId;
    const type = req.query.type as string;
    const workspaceId = req.query.workspaceId as string;
    
    const filter: any = { userId: userId };
    
    if (type) {
      filter.type = type;
    }
    
    if (workspaceId) {
      filter.workspaceId = workspaceId;
    }

    const content = await contentModel
      .find(filter)
      .select('title link content type tags workspaceId createdAt updatedAt')
      .populate("userId", "username");

    res.json({ content })
  } catch (error) {
    return res.status(500).json({
      message: "Internal Server Error"
    });
  }
});
```

### Frontend type filtering

The Dashboard applies type filters client-side:

```typescript
function handleFilterSelect(type: string) {
  setFilter(prev => (prev === type ? null : type));
}

const displayed = dataToDisplay.filter((item: any) => {
  const matchesType = !filter || item.type === filter;
  const matchesWorkspace = !selectedWorkspace || item.workspaceId === selectedWorkspace;
  return matchesType && matchesWorkspace;
});
```

## Combining filters

### Multiple filter combinations

You can combine type and workspace filters:

```typescript
// All notes in a specific workspace
GET /api/v1/content?type=note&workspaceId=65abc123

// All YouTube videos in a workspace
GET /api/v1/content?type=youtube&workspaceId=65abc123
```

## Tag-based organization

### Tag schema

Content can have multiple tags:

```typescript
tags: [{type: mongoose.Types.ObjectId, ref: 'Tag'}]
```

### Adding tags to content

```typescript
const createdContent = await contentModel.create({
  title: "React Best Practices",
  type: "note",
  content: "# Best Practices\n...",
  tags: req.body.tags || [],
  userId: req.userId,
});
```

<Note>
  Tags are included in the text search index, making tagged content discoverable through search.
</Note>

## Best practices

### Effective searching

1. **Use specific terms**: More specific queries return more relevant results
2. **Try variations**: Search for different forms of keywords
3. **Combine filters**: Use type and workspace filters to narrow results
4. **Use tags**: Tag content consistently for better organization

### Organizing for searchability

```typescript
// Good: Descriptive title and content
{
  title: "React useEffect Hook Explained",
  content: "The useEffect hook handles side effects in React...",
  tags: ["react", "hooks", "tutorial"]
}

// Less searchable: Vague title
{
  title: "Notes",
  content: "Some stuff about hooks",
  tags: []
}
```

<Tip>
  Use descriptive titles and add relevant tags to make your content easier to find later.
</Tip>
